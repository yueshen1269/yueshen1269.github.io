设计模式

起源-建筑学

定义：

在面向对象软件设计过程中针对特定问题的简介而优雅的解决方案，通俗的说就是在某种场合下对某个问题的一种解决方案，给一些好的设计去个名字。

学习模式的作用：

软件设计中，模式是一些经过了大量实际项目经验验证的优秀解决方案，熟悉这些模式的程序员，对某些模式的理解也许形成了条件反射。当合适的场合出现时，可以很快找到某种模式作为解决方案。

模式在不同语言之间的区别

设计模式的适用性

设计模式确实有可能带来代码量的增加，也许会把系统的逻辑搞得更复杂，但设计模式的作用是让人们写出可复用和可维护性高的程序。

所有的设计模式的实现都遵循一条原则，即“找出程序中变化的部分，并将变化封装起来”。程序中变化的部分封装起来，剩下的不变和稳定部分是非常容易复用的。

注意：不能误用和滥用，将一些模式用在了错误的场景中，或者不该使用模式的地方刻意使用模式。

模式使用正确的地方：要深刻理解了模式的意图之后，在结合项目的实际场景才会知道

分辨模式的关键是意图而不是结构

辨别模式的关键是这个模式出现的场景，以及为我们解决了什么问题。

对JavaScript设计模式的误解

  1. 习惯把静态类型语言的设计模式照搬到JavaScript中
  2. 习惯根据模式的名字去臆测该模式的一切

模式的发展

某种解决方案要成为一种模式，要遵守几个原则：

  • 再现
  • 教学
  • 能以一个名字来描述这种模式
